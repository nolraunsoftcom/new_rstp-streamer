# M4 — MediaMTX Relay 통합 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development 으로 태스크 단위 실행. 단계는 체크박스(`- [ ]`).

**Goal:** 장비와 viewer 사이에 MediaMTX relay를 두어 (1) 장비 보호막(장비당 상시 1연결), (2) 외부 다중 PC 시청 준비, (3) Control API 기반 "장비→Relay 수신" 진단을 제공한다. viewer는 relay를 자식 프로세스가 아닌 **OS 서비스**로 관리한다.

**Architecture:** 클린아키텍처 유지. 순수 `RelayConfig`(domain) + `RelaySupervisor`(app, 4역할) + 포트 2종(`IRelayServiceManager`, `IRelayControlApi`) + 플랫폼 어댑터(infra). 진단 모델은 이미 M1에 예비됨(`HealthStage::RelayIntake`, `DiagnosisReason::RelayDown/RelayNoSource`, `ConnectionStateMachine::sourceAvailableHint` + `ChannelController::notifySourceAvailable()`). 채널은 relay/직결 모드 선택.

**Tech Stack:** C++20, Qt6(Widgets + **신규 Qt::Network** for Control API HTTP), FFmpeg, MediaMTX **v1.19.1 고정**, Catch2. **플랫폼: Windows 메인**(서비스=Windows SCM), macOS는 개발/실기기 테스트(LaunchAgent). 장비는 이 맥에 link-local(169.254.4.1) 연결 — 실기능 검증은 맥, Windows 어댑터는 별도 Windows 환경 검증.

**플랫폼 주의(중요):** 사용자 1순위 배포 타깃은 **Windows**. 따라서 `IRelayServiceManager`의 Windows 어댑터를 1급으로 구현(스텁 아님)하되, 이 세션은 맥이라 자동 검증은 macOS 어댑터로 한다. Windows 어댑터는 컴파일·로직 리뷰까지만 보장하고 실행 검증은 Windows 박스 + 장비로 별도 수행(M4 수용 잔여 → 별도 단계). "클라이언트가 쉽게"=설치 시(M5 NSIS) 서비스 1회 등록, 런타임엔 viewer가 자동 감지·기동 요청.

**범위(이번 이터레이션, 사용자 확정):** 기능완성 + 단기검증. RelaySupervisor 4역할 + 진단 통합 + `sourceOnDemand:no` 검증 + viewer 재시작 시 장비 세션 무중단 + 수시간 소크. **72h 소크 + 전원차단 주입은 별도 수용 단계.**

- [ ] **Step 1: 실패 테스트** — `RelayConfig::generate(channels)`가 relay 채널마다 path 블록(`source: <deviceUrl>`, `sourceOnDemand: no`, `rtspTransport: tcp`)을 포함하고, 직결 채널은 제외하며, 전역 `rtspAddress: 127.0.0.1:8554`/`apiAddress: 127.0.0.1:9997`를 포함하는지 검증.
```cpp
TEST_CASE("RelayConfig: relay 채널만 path 생성, sourceOnDemand:no/tcp 하드코딩") {
    using namespace nv::domain;
    std::vector<RelayPath> ch{
        {"ch1","rtsp://169.254.4.1:8900/live", true},   // relay
        {"ch2","rtsp://169.254.4.1:8901/live", false},  // 직결 — 제외
    };
    const std::string yml = RelayConfig::generate(ch);
    CHECK(yml.find("rtspAddress: 127.0.0.1:8554") != std::string::npos);
    CHECK(yml.find("apiAddress: 127.0.0.1:9997") != std::string::npos);
    CHECK(yml.find("rtspTransports: [tcp]") != std::string::npos);
    CHECK(yml.find("ch1:") != std::string::npos);
    CHECK(yml.find("source: rtsp://169.254.4.1:8900/live") != std::string::npos);
    CHECK(yml.find("sourceOnDemand: no") != std::string::npos);
    CHECK(yml.find("ch2:") == std::string::npos);   // 직결 제외
    // 검증 함수: 생성물이 하드룰 만족
    CHECK(RelayConfig::validate(yml).ok);
}
TEST_CASE("RelayConfig: sourceOnDemand가 yes면 validate 실패(보호막 무효)") {
    auto r = nv::domain::RelayConfig::validate("paths:\n  ch1:\n    sourceOnDemand: yes\n");
    CHECK_FALSE(r.ok);
}
```
